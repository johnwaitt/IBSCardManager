# Release Notes

## v2.2.0
### Highlights
- AI Knowledge Engine Core Phase 1 implemented with knowledge records, evidence tracking, correction tracking, decision history, deterministic confidence scoring, and safe-learning guardrails.
- Scanner workflow now records foundational decision/evidence/correction metadata for future intelligence phases.
- Knowledge Review Queue and Knowledge Record detail surfaces added for review and diagnostics workflows.
- Diagnostics and backup manifest expanded for knowledge schema/rule/model/version and health visibility.

### Versioning
- Authoritative application version updated to **2.2.0**.
- Upper-left application label displays **Ver 2.2.0** using runtime version provider.
- Application version remains separate from schema, catalog, knowledge schema, model, and rule versions.

## v2.0.1
### Highlights
- Master Database compatibility addendum implemented for future v2.1.0 architecture.
- Catalog abstraction boundaries introduced for provider-swappable catalog access.
- Catalog-readiness integrity checks and backup manifest readiness added.
- Non-destructive schema and identifier-preservation strategy enforced.

### Versioning
- Authoritative application version updated to **2.0.1**.
- Upper-left application label displays **Ver 2.0.1**.

## v1.7.0
### Highlights
- Collection-wide analytics and KPI dashboard enhancements.
- Grading ROI engine with expected value, break-even grade, and recommendation confidence.
- Sell-now and hold recommendation scoring with explainable reasons/risks.
- Smart recommendation records with accept/dismiss/snooze state tracking.
- Collection value snapshots and trend history support.
- CSV/JSON export support for analytics reports.

### Versioning
- Authoritative application version updated to **1.7.0**.
- Footer and About surfaces now display runtime build version.
